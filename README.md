# NicoRepo2

NicoRepo2 は、ニコニコ動画上のユーザー行動を検知し、旧ニコレポ Twitter 連携のように X へ自動で個別投稿する Chrome / Edge 向け Manifest V3 拡張機能です。

Chrome Web Store での公開は前提にしていません。開発者モードで「パッケージ化されていない拡張機能」として読み込んで使います。

## 機能

- ニコニコ動画の watch ページから動画タイトル、URL、動画 ID、タグ、投稿者名を取得します。
- 初期状態では動画タグに関係なく投稿します。
- 対象タグはフィルタ機能やカスタムテンプレート変数用です。初期値は `音MAD` ですが、初期投稿文には入りません。
- options 画面で「対象タグに一致する動画だけ投稿する」を ON にした場合だけ、対象タグで投稿対象を絞り込みます。
- いいね、マイリストのクリックを検知します。
- 履歴、フォロー新着、ランキングなど watch ページ以外のマイリスト登録は、登録APIの成功通信を検知して投稿します。
- 行動 1 件につき X 投稿 1 件を生成します。
- 同一動画 ID かつ同一行動の重複投稿を防止します。
- X API は使わず、ログイン済みブラウザ上の X Web 投稿画面を操作します。
- 投稿イベントごとに一時的な popup ウィンドウを背面に開き、投稿完了後に閉じます。
- X自動投稿は初期状態で1日25件までです。options画面から1〜1000件の範囲で変更できます。
- いいね・マイリスト投稿では、ニコニコ本体の未ログイン相当レスポンスが `HARMFUL_VIDEO` の場合だけ、投稿URLを `nicovideo.gay` に置き換えられます。
- 成功ログ、失敗ログ、投稿済みキー、未処理キューを `chrome.storage.local` に保存します。
- options 画面から対象タグ、行動ごとの ON/OFF、テンプレート、ログを管理できます。
- スマホ連携として、ニコニコのログイン済みアカウントページを定期確認し、スマホ側で増えたいいね/マイリスト動画を自動回収できます。

ニコニ広告の自動投稿は未対応です。誤検出を避けるため、現在の設定画面には表示していません。

## インストール方法

1. Chrome または Edge で `chrome://extensions/` を開きます。
2. 右上の「デベロッパーモード」を ON にします。
3. 「パッケージ化されていない拡張機能を読み込む」を押します。
4. このリポジトリ内の `NicoAutoReport` フォルダを選択します。

Edge の場合は `edge://extensions/` から同じ手順で読み込めます。

## 使い方

1. ニコニコ動画にログインしておきます。
2. X にログインしておきます。
3. 拡張機能の options 画面で、対象タグと投稿テンプレートを確認します。
4. `https://www.nicovideo.jp/watch/*` の動画ページを開きます。
5. いいねまたはマイリストをクリックします。
6. 拡張機能が X の intent 投稿画面を popup ウィンドウで開き、投稿欄へ本文を入れて投稿ボタンをクリックします。
7. 投稿成功または失敗の結果は options 画面のログに保存されます。

1日の上限に到達した場合、残りのイベントは失敗として削除せずキューに保持します。PCのローカル日付が変わると自動的に処理を再開します。上限カウンターはログとは別に保存されるため、ログを削除しても当日の投稿数はリセットされません。

## スマホ連携

スマホアプリ上の操作をChrome拡張機能が直接検知することはできないため、PC側でニコニコのログイン済みアカウントページを定期確認します。

1. PC側でもニコニコにログインしておきます。
2. 「スマホ連携」は初期状態で ON です。不要な場合は options 画面で OFF にできます。
3. 通常はそのまま保存します。確認間隔と最大件数だけ詳細設定で変更できます。
4. 設定保存後とChrome起動後に自動確認します。
5. 初回同期は現在の一覧を基準として保存するだけで投稿しません。
6. 次回以降、取得元ページに増えた動画を確認し、短時間後にもう一度残っていることを確認してから既存キューへ追加します。
7. 既存の重複防止、デバイス規制URL置換、X自動投稿、ログ保存の処理に流れます。

いいね履歴はニコニコのログイン必須APIを自動で使います。マイリストはログイン中ユーザーのマイリスト一覧から公開マイリストだけを自動発見し、各マイリストを確認します。非公開マイリスト、または公開状態を確認できないマイリストは投稿しません。

### 端末・会員差への対策

- 一般会員・プレミアム会員で公開状態の返却形式が異なっても、`isPublic`、`isPrivate`、公開範囲文字列など複数形式を判定します。
- 個別マイリストAPIが利用できない環境では、ログイン中ユーザーのマイリスト一覧APIへフォールバックします。
- 複数の公開マイリストを持つ場合も、特定の大きいマイリストだけで取得上限が埋まらないよう、各リストの直近項目を確認します。
- Chrome起動、設定変更、定期アラームが重なっても、スマホ同期は同時に複数実行しません。
- ニコニコ側が一時的に空の一覧を返しても、確認待ちの候補は30分保持して次回確認へ回します。
- 新しく公開されたマイリストは、公開前から入っていた動画を投稿せず、現在内容を基準化してから新規追加だけを検知します。
- 同期間隔内に2件の操作があっても、初期設定の最大回収件数内であれば両方を順番に処理します。

拡張機能の保存データはブラウザプロファイルごとに独立しています。別PCや別ブラウザへ導入した直後は、その環境で初回基準化が必要です。また、そのブラウザでニコニコとXの両方にログインしている必要があります。

日次投稿上限もブラウザプロファイルごとに管理されます。同じXアカウントを複数PC・複数プロファイルから利用した場合、各環境の投稿数を合算することはできません。複数環境で同時利用する場合は、それぞれの上限を低めに設定してください。

## 初期投稿テンプレート

いいね:

```text
【いいね！】{title}
{url}
#{videoId} #ニコニコ動画
```

マイリスト:

```text
【マイリスト】{title}
{url}
#{videoId} #ニコニコ動画
```

利用できる変数:

- `{title}`
- `{url}`
- `{videoId}`
- `{targetTag}`
- `{targetTagHash}`
- `{authorName}`
- `{actionLabel}`

## 保存データ

すべてのデータは `chrome.storage.local` に保存されます。

- `settings`
- `postedKeys`
- `postedLogs`
- `failedLogs`
- `queue`

ログイン情報、Cookie、パスワード、認証情報は保存しません。

## X API と外部通信について

この拡張機能は X API を使用しません。

外部サーバー通信も行いません。投稿処理は、ユーザー本人がログイン済みのブラウザ上で X の Web 投稿画面を開き、content script で投稿欄と投稿ボタンを操作する方式です。

デバイス規制らしい動画のURL置換は、ニコニコ本体への未ログイン相当の確認結果だけを使います。拡張機能から `nicovideo.gay` へ直接アクセスすることはなく、投稿本文内のURLだけを差し替えます。

## 注意事項

- X の DOM は頻繁に変わるため、投稿欄や投稿ボタンの検出は将来的に調整が必要になる可能性があります。
- 対応対象はデスクトップ版の最新Chrome / Edgeです。スマホアプリの操作は、PC側のChrome / Edgeが起動している間の定期同期で回収します。
- 短時間に多数の投稿テストを行うと、X 側で投稿制限、ラベル、表示制限などの対象になる可能性があります。実アカウントでの連続テストは避けてください。
- 初期上限25件はXによる安全保証ではありません。アカウント作成直後や同じ形式のリンク投稿が続く場合は、さらに低い値を推奨します。
- ログイン画面、CAPTCHA、追加認証、アカウントロック、制限画面を検知した場合は投稿を中止します。突破処理は行いません。
- 投稿成功判定が取れない場合は失敗ログに記録します。
- 失敗時の無限再試行は行いません。
- 完全自動投稿を行う拡張機能のため、利用するアカウントや投稿頻度には十分注意してください。
